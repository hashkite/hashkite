Setting Up Drupal with DDEV and Docker on macOS

  1. Home
  2. Blog
  3. Setting Up Drupal with DDEV and Docker on macOS

DDEV (development environment for Drupal 8/9) offers a convenient way to set up a local Drupal development environment using Docker containers. Here's a guide to get you started on macOS:

Prerequisites

  • Mac computer: Any recent Mac model will work.
  • Docker: Download and install Docker Desktop for Mac from [https://www.docker.com/products/docker-desktop].
  • ddev CLI: Install ddev using Homebrew:
    ```bash brew install drud/ddev-edge/ddev ```

1. Create a Drupal Project Directory

Open your terminal and navigate to your desired project directory. Create a new directory for your Drupal project (e.g., `mydrupalsite`).

2. Initialize the DDEV Project

Navigate to your project directory using the `cd` command (e.g., `cd mydrupalsite`). Then, run the `ddev config` command to initialize a DDEV configuration file:

ddev config

Follow the interactive prompts to choose your web server (Apache or Nginx), PHP version, and database (MySQL or MariaDB).

3. Start the Drupal Service

With the configuration complete, run `ddev start` to initiate the containers and set up your Drupal environment.

ddev start

4. Accessing Your Drupal Site

Once the containers are up and running, you can access your Drupal site at the following URL by default (replace `` with your project name):

http://<ddev-project-name>.ddev.site

The default username and password for the Drupal administrator account will be displayed in the terminal output during startup.

5. Building and Developing

Your Drupal codebase will be mounted inside the container, allowing you to make edits on your local machine and see the changes reflected in the browser immediately. Use your preferred code editor or IDE to work on your Drupal project files.

Additional Considerations

  • DDEV offers various commands for managing your development environment. Explore the DDEV documentation for more details: [https://ddev.readthedocs.io/en/latest/]
  • You can install Drupal core and contributed modules directly within the container using Drush or Composer.

Benefits of Using DDEV

  • **Fast and Easy Setup:** DDEV simplifies the process of setting up a local Drupal environment compared to manual configuration.
  • **Isolation and Consistency:** Each project has its own isolated container, ensuring consistency across development environments.
  • **Simplified Management:** DDEV provides commands for starting, stopping, and managing your development environment efficiently.
